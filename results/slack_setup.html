🛠 Slack 設定手順（所要時間 3分）

1ブラウザで Slack API を開く
まずブラウザ（iPhone/MacどちらでもOK）で以下のURLにアクセス:
https://api.slack.com/apps
Slackにログインしていない場合はログインしてください。
2新しいアプリを作成
緑色の「Create New App」ボタンをクリック。
表示されたダイアログで「From scratch」（ゼロから作成）を選択。
  • App Name: 気持ちマックス などお好きな名前
  • Pick a workspace: 通知を受けたいワークスペースを選択
「Create App」をクリック。
3Incoming Webhooks を有効化
左側メニューの「Incoming Webhooks」をクリック。
画面右上のトグルスイッチを「On」に切り替え。
4通知チャンネルを選んで Webhook URL を発行
ページ下部の「Add New Webhook to Workspace」ボタンをクリック。
通知を送りたいチャンネルを選択（例: #投資通知 など好きなチャンネル）。
「許可する」をクリック → Webhook URLが発行されます。
⚠️ 重要: このURLは秘密です。外部に公開しないでください。 このURLを知っている人は、あなたのSlackチャンネルにメッセージを送れます。
5Webhook URL をコピー
発行されたURLの右にある「Copy」ボタンをクリック。
以下のような形式です:
https://hooks.slack.com/services/T0XXXX/B0XXXX/XXXXX...
6ターミナルで設定コマンドを実行
MacBookのターミナル（Terminal.app）を開き、下記コマンドをコピペして実行:
python3 /Users/sanosano/projects/kimochi-max/discord_notify.py setup
URL貼り付けを求められたら、Step 5でコピーしたSlack Webhook URLを貼り付けて Enter。
✅ 設定成功すると、すぐにテスト通知がSlackの指定チャンネルに届きます。
7ボットを再起動（通知を有効化）
以下のコマンドで、通知が有効なボットに再起動:
pkill -f demo_runner.py && cd /Users/sanosano/projects/kimochi-max && nohup python3 demo_runner.py > /tmp/kimochi_demo.log 2>&1 &
これで次のトレード時・DD警告時・日次サマリー時にSlackに通知が届きます🎉

💡 ポイント

📱 iPhone/Androidで通知を受け取る:
Slack公式アプリをスマホにインストールして、通知設定で「すべての新規メッセージ」にしておけば、トレード発生時に即PUSH通知が届きます。
🔕 特定時間帯は通知OFF:
Slackには「おやすみモード」があるので、深夜は通知を止める等の設定が本体側でできます。
👥 複数人でシェア:
チームのチャンネルに通知を流せば、複数人で監視できるのもSlackの強み。

❓ よくある質問

Q. Slackワークスペースを持っていません
Slackの公式サイトから無料でワークスペースを作成できます（メールアドレスのみで作成可能）。 個人利用でも問題ありません。
Q. 通知が届きません
まず以下のコマンドでテスト送信:
python3 /Users/sanosano/projects/kimochi-max/discord_notify.py test
テスト通知が届かない場合:
  • Webhook URLが正しいか: status コマンドで確認
  • Slackアプリの権限: Incoming Webhooks が無効化されていないか
  • 選択したチャンネル: 実は別チャンネルに投稿されていないか
Q. Discord・Slack両方に通知したい
現在は1つのWebhook URLのみ対応していますが、両方に通知したい場合は設定ファイルを複数URL対応に拡張できます。 必要であれば実装しますのでお知らせください。
