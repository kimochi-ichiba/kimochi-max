# iter71: v2.4 トレイル閾値の Walk-Forward 検証 (2020-2025)

## 概要

PR #8 (v2.4) で導入された `dynamic_regime` + `trail_stop_ach 30%` + `trail_stop_btc 20%` が
W4 期間 (1.3 年) 以外の相場でも有効か、Walk-Forward 4 窓で検証した。

- 期間: 2020-01-01 〜 2025-04-19 (キャッシュ `_iter61_cache.pkl` の全範囲)
- ユニバース: 62 銘柄
- パラメータ格子: 4 × 4 × 4 = 64 セル × 4 窓 = **256 バックテスト実行**

## 主要な発見

### 1. PR #8 の「+117% CAGR」の出どころ

PR #8 本文の「W4 OOS (2024-01〜2025-04-19, 1.3 年) で CAGR +117%」を分解すると:

- **2024 単独 (W3 OOS)**: 総リターン **+270%** ← ブル相場の爆発力
- **2025 Q1 (W4 OOS)**: 総リターン **-45%** ← トレイル whipsaw による損失

両者を連結した PR #8 の 1.3 年では +117% に均される構造。ブル 2024 に強く寄与し、
2025 Q1 の失速は数字上は見えにくくなっている。

### 2. ベア相場 (2022) では健全

W1 OOS で **+5.9% / DD 20.6%** (PR #8 既定値)。ベア相場で「生存」どころか
プラスで抜けている。これは以下の Bear 即時退避機構が機能したため:

- BTC < EMA200 検知 → ACH 全売却
- `dynamic_regime` により ACH cash を USDT へ完全退避
- 結果: 2022 年のほとんどを USDT 待機で過ごす (Bear 退避・Trail 発動いずれも 0 回)

### 3. トレイル閾値の過学習は「軽度」

- IS Calmar 最良セルが全 4 窓で:
  - `trail_stop_ach`: W1〜W3 で **0.25** が最良、W4 のみ 0.30
  - `trail_stop_btc`: **0.20 または 0.30** で割れる
  - `bull_ach_weight`: **0.50 または 0.55** で割れる (PR #8 の 0.60 は選ばれず)
- ただし PR #8 既定の `0.30/0.20/0.60` と本検証最良の `0.25/0.20/0.50` の差は限定的で、
  汎化性能はどちらも同等レベル。

### 4. 2025 Q1 (W4 OOS) の失速はトレンドフォローの構造的弱点

- 3.5 ヶ月で総リターン -43〜-45%、トレイル発動 ACH 4 回 / BTC 1 回
- チョピー相場で「peak → -20% → 売り」を繰り返し、**whipsaw (往復ビンタ)** で削られた
- これは v2.4 特有の欠陥ではなく、トレンドフォロー全般の既知の弱点

## 判定

- ✅ **ベア相場 (2022) での生存**: v2.4 は採用可能なレベル
- ✅ **過学習は軽度**: PR #8 既定値は最適ではないが、近傍にあり差は小さい
- 🟡 **chop 相場での脆弱性**: 2025 Q1 のような持ち合い相場で損失を避けられない
  → トレイルを緩める (閾値を広げる) か、volatility filter を追加する余地あり
- 🟡 **CAGR の派手さに注意**: PR #8 の +117% は 2024 単独の +270% に引っ張られた結果

## PR #8 への推奨

- **マージは可**（ただし慎重に）: ベア生存は確認、過学習は軽度
- **実資金投入はまだ早い**: 2025 Q1 のような失速パターンが SIM でも再現。
  現状の SIM モード観察継続が正しい判断
- **次の改善テーマ**:
  - 追加 filter: ATR/ADX で chop 相場を検出してトレイル閾値を動的に広げる
  - 部分決済: 全売却ではなく 50% 決済に変えて whipsaw 耐性を上げる
  - DD 目標を 55% → 30% に絞った上での再探索
