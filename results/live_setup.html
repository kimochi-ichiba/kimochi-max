🔐 気持ちマックス 実取引 (LIVE) セットアップ

SIMモード（仮想）から LIVEモード（実資金）への切替ガイド。 実資金が動くため、手順を厳密に守ってください。

⚠️ 最重要：実資金は失う可能性があります

  • 過去のバックテストは将来を保証しません。未来の相場は予測不能
  • DD 40-50% は確実に発生する可能性があります（資産半減の覚悟必要）
  • 取引所倒産・ハッキング・APIキー流出 のリスクあり
  • 最初は $50〜$100 程度の最小額 で1週間以上テストすることを強く推奨
  • 設定・コードに不具合があれば、想定と違う発注をする可能性あり
  • この機能によって生じた損失は、全てご自身の責任となります

📊 3つの動作モード

🟢 SIM
仮想のみ
現在のデフォルト。
仮想$10,000でシミュレーション。
実資金は一切動きません。
🟡 DRY_RUN
シグナル確認
APIキー設定済みだが
LIVE_ENABLED未設定。
シグナルは出るが発注しない。
🔴 LIVE
本番発注
全設定揃ったとき。
実資金が動きます。
1注文$100・1日5回上限。

🛠 セットアップ手順 (段階的・所要時間30分)

1Binance にログインしてAPIキーを作成
  • Binance (binance.com) にログイン
  • 右上のアカウントアイコン → API Management
  • 「Create API」→ 名前「KimochiMax」で作成
  • 2FA認証 (メール + Google Authenticator等) を完了
2APIキーの権限を【必須】制限する
APIキーに「出金 (Withdraw)」権限を絶対に付けないでください
  • ✅ Enable Reading (残高確認用、ON)
  • ✅ Enable Spot & Margin Trading (取引用、ON)
  • ❌ Enable Withdrawals (OFF 絶対必要)
  • ❌ Enable Futures (必要に応じて。初期はOFF推奨)
🔐 IP制限の推奨: 「Restrict access to trusted IPs only」をONにして、 あなたのMacBookの現在IPを登録。仮にキーが流出しても別IPから使えない。
3APIキーをターミナルに登録 (環境変数)

MacBookのターミナルで以下を実行（YOUR_API_KEY等を置き換え）:

cat >> ~/.zshrc << 'EOF' # 気持ちマックス Live取引設定 export BINANCE_API_KEY='YOUR_API_KEY' export BINANCE_SECRET='YOUR_SECRET' export KM_MODE='live' # 安全装置: 以下を 1 にすると本番発注が有効 # export LIVE_ENABLED='1' # 取引上限 (任意で変更) export KM_MAX_ORDER_USD='100' export KM_MAX_DAILY_TRADES='5' EOF source ~/.zshrc
⚠️ LIVE_ENABLED はまだコメントアウトしておいてください。 DRY_RUN モードで1週間テストしてから有効化します。
4API接続テスト (残高確認のみ)
python3 /Users/sanosano/projects/kimochi-max/live_trader.py check

以下が表示されれば成功:

✅ 接続成功
APIキー権限: TRADE
💰 残高: USDT free=100.00
📊 本日の実取引回数: 0/5

❌「WITHDRAW権限あり」と出たら、Binanceコンソールで権限を無効化してください。

5DRY_RUNモードで1週間テスト
launchctl kickstart -k gui/$(id -u)/com.sanosano.kimochimax.demo tail -f /tmp/kimochi_demo.log | grep "DRY_RUN"
  • 実発注せず、シグナルだけ出る「DRY_RUN」モードで動作
  • 1週間観察して、発注指示が想定通りのタイミングで出るか確認
  • Discord通知でBUY/SELL予告が届く
6少額($50〜$100)で LIVE を有効化
⚠️ 最初は必ず少額でテストしてください。いきなり大額を入れると損失時に取り返しがつきません。
  1. Binance Spot にUSDT $50〜$100 程度を入金
  2. ~/.zshrc の export LIVE_ENABLED='1' のコメントを外す
  3. ターミナル再起動 or source ~/.zshrc
  4. ボット再起動: launchctl kickstart -k gui/$(id -u)/com.sanosano.kimochimax.demo
  5. 起動ログで「🔴 取引モード: LIVE」が出ていることを確認
  6. 次のBUYシグナル発動時に実発注される（Binanceの取引履歴で確認）
7モードの無効化 (取引停止)
unset LIVE_ENABLED # または ~/.zshrc からLIVE_ENABLEDの行をコメントアウト launchctl kickstart -k gui/$(id -u)/com.sanosano.kimochimax.demo

これで即座に DRY_RUN に戻り、発注がストップします（既存ポジションは維持）。

🛡️ 組み込み安全装置

✅ APIキー2つ揃わないと LIVE は動かない (BINANCE_API_KEY + BINANCE_SECRET)
✅ LIVE_ENABLED=1 明示的設定が必須 (デフォルトは DRY_RUN 止まり)
✅ 1注文 $100 上限 (KM_MAX_ORDER_USD)
✅ 1日 5回 取引上限 (KM_MAX_DAILY_TRADES)
✅ サーバー時刻ズレ1秒超で拒否 (Mac時刻異常を検出)
✅ 発注失敗時 SIMフォールバック (エラー時も継続稼働)
⚠️ APIキー出金権限は絶対OFF (あなたが設定すること)
⚠️ IP制限推奨 (Binance側でトラステッドIP限定)

❓ よくある質問

Q. APIキー流出したらどうなる？

出金権限OFF + IP制限ONなら、流出しても資金は動かせません（取引のみ可）。 発見次第、Binanceコンソールで該当APIキーを即削除してください。

Q. 夜中に大きな下落で清算される？

Spot取引のため「清算」は発生しません（レバレッジ無し）。 最悪でもBTC価格下落分の含み損で、強制決済にはなりません。 Futures/Marginは本バージョンでは対象外です。

Q. Mac再起動後は？

launchdが自動起動。ただし環境変数 LIVE_ENABLED は ~/.zshrc 経由で 読み込まれるため、launchd起動時には設定されていない可能性あり。 plistに環境変数を直接書くか、健全なDRY_RUN運用を推奨。
