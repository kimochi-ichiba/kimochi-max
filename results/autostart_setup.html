🚀 H11Bot 自動起動 セットアップ

Mac を再起動しても H11Bot が自動的に動き続けるように設定します（macOS launchd を使用）

💡 これで何が解決される？

現在、Mac を再起動すると以下が停止します:

  • 📊 Web サーバー (http://localhost:8080/)
  • 🤖 デモランナー (BTC価格の5分毎チェック)
macOS launchd は、システム起動時に自動でプログラムを動かす仕組みです（Linuxの systemd 相当）。 設定すれば、MacBookを起動するだけで自動的に H11Bot が動き始めます。 スリープから復帰した時も自動で再起動します。

🛠 セットアップ手順（所要時間 30秒）

1ターミナルを開く
MacBook の ターミナル（Terminal.app）を開きます。
Cmd+Space で「ターミナル」と検索すれば開けます。
2インストールコマンドを実行
以下のコマンドをコピーしてターミナルに貼り付け、Enter:
bash /Users/sanosano/projects/kimochi-max/launchd/install.sh
実行結果:
  • ✅ ~/Library/LaunchAgents/ に plist を配置
  • ✅ launchctl にサービス登録
  • ✅ Web サーバー + デモランナーを起動
3動作確認
ブラウザで以下を開いて、ダッシュボードが見えれば成功:
http://localhost:8080/demo.html
✅ 今後はMacを再起動しても、自動的に H11Bot が動き続けます🎉

🔧 管理コマンド

📊 サービス状態確認
launchctl list | grep h11bot
🔄 デモランナー再起動
launchctl kickstart -k gui/$(id -u)/com.sanosano.h11bot.demo
📄 ログ確認
tail -f /tmp/kimochi_demo.log
tail -f /tmp/kimochi_server.log
🛑 自動起動を無効化（アンインストール）
bash /Users/sanosano/projects/kimochi-max/launchd/uninstall.sh

❓ よくある質問

Q. MacBookをシャットダウンしたらどうなる？
シャットダウン中は停止します（PCが動いていないので当然）。
次回 MacBook を起動（ログイン完了）した時に、自動で復帰します。
Q. スリープ中も動きますか？
スリープ中は停止します（MacBookの電源設定に従います）。
スリープさせたくない場合は:
システム設定 → ディスプレイ → 詳細設定 →
「ディスプレイがオフのときコンピュータを自動でスリープさせない」を ON にしてください。
Q. エラーが出たらどうする？
ログを見ると原因が分かります:
tail -50 /tmp/kimochi_demo.log
よくある原因:
  • Python not found: Python3がインストールされていない
  • port already in use: 別のプロセスがポート8080使用中（lsof -ti:8080 | xargs kill -9で解放）
  • permission denied: chmod +x で実行権限付与
Q. MacBookを売却する時は？
売却前に uninstall.sh を実行すれば、自動起動設定を削除できます。
プロジェクトディレクトリも一緒に削除すれば完全に跡は残りません。
